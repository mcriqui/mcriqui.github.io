Challenge

The main project for my summer internship was to work on multiple parts of Ricoh's Job Ticket Editor. It's a component in many of Ricoh's Production Print software products that allows the user to edit and make changes to a print job. The field identified many key areas for improvement. I worked with my mentor to improve multiple sections of this software.

Header and Footer

One of the more challenging areas I worked on was header and footer. In this area, print shop operators can add, edit and remove headers and footers. The previous version was confusing for users for the following reasons:

Idea Generation

Early on, I generated many sketches, trying to think of as many possible ways to represent header and footer. I then worked with my mentor to get her input and identify directions that had the most potential. Below are examples of my design directions.

The user selects the position and adds the date or time into that position.

The add button is replaced by a plus sign indicating that the user adds the date or time to the position

Similarly to Adobe Acrobat, the user clicks the button to insert the date and time.

Images represent the position of the header or footer. The user adds or deletes a header or footer using the addition and subtraction buttons.

The user does not typically delete a header or footer so the functionality is removed.

Competitive Analysis

For inspiration, I looked at how other software applications manage headers and footers.

Google Drive

Drive allows you to edit directly in the document. While this made it easy for the user to change the header and footer and get a clear idea of how it would look, this isn’t possible with the current JTE software.

Adobe Acrobat

Adobe Acrobat’s editing is closely aligned with JTE. A strength is that the 6 boxes match with a user’s mental model of where the header and footer would be positioned on the page.

It also has many options for date and time format. This is a glaring weakness in JTE’s current design. It’s very confusing to set the date and time. Acrobat has a much more intuitive and clear way to insert date, time and page numbers. Separating all these options makes it easier for the user to know what the are changing.

Microsoft Word

Similarly to Google Drive, Microsoft allows the user to edit the header and footer directly in the document. A key strength is their use of icons to visually communicate various actions.

Review with Developers

After my competitive analysis, I made some final changes to my design and presented the design below to the developers on the project. Here, a user selects the position and that position is highlighted in blue to clearly indicate where she is adding the date and time. A user can then select a date or time format and add that to the highlighted position. The intention was to make it clearer and more obvious for how you insert a header and footer.

I quickly learned that what I proposed was very difficult at a technical level and the developers were less than thrilled about the prospect of implementing my proposed design.

 

While this was initially a setback, it was important because I was able to call their attention to a key usability issue with the header and footer area. They were also able to share their expertise about how the back end works. We were able to jointly brainstorm ideas with a better understanding of each other's perspectives.

Iteration

With input from the developers and the help of my mentors, I went back to the literal drawing board and reworked my design. I was able to reach a compromise which both the developers and UX team agreed on. I also had to make adjustments to my design to make sure that my UI patterns were in line with the team’s patterns to ensure consistency among the Ricoh Production Print software products.

The updated interface has a cleaner look leveraging whitespace to make the interface easier for the user to interact with. The words 'left', 'center' and 'right' are an addition so the user clearly knows what she is editing.

When the user selects a position, it becomes highlighted in blue. If the user wants to type in a string, she can type directly in the entry field.

If the user wants to add a date, time, page number or saved string, she can do so by selecting the option from the drop down menu. Previously, the drop down menu only contained previously inputed strings.

Adding the date, time and page number in the drop down doesn't require the user to memorize random confusing symbols. Here she selects date, so the word date and symbol are populated in the entry field.

If the user wants to change the date or time format, she can do so by choosing it from the drop down menus. A future strategy is to populate the chosen format in the position entry but we were not able to implement that in this release.

Reflection

I had two major takeaways from this project.

 

Business Constraints

The first was how to work in industry with real business constraints. Unlike starting from scratch, there were a lot of important considerations in dealing with legacy software. I also had to think about the actual implementation of the design which was something new to me. Additionally, I had to work with many key stakeholders, including users, product owners, developers and other UX designers. It helped me learn how to meaningfully incorporate everyone's opinions and viewpoints into my designs.

 

 

Usability Test Environment

Secondly, a key takeaway was the importance of doing usability testing in the environment in which the software will be used. When I was designing, I was thinking my users would give their full attention to the software. When I got to one of the print shops, the first thing I noticed was how loud it was with all the printers. Then, during the test, the user was constantly looking back and forth between the test and his printers running to check on the print jobs. The was important for me to see and realize I needed to make my designs as easy to use as possible
